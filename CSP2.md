## Christmas Shop. Часть 2 - Адаптивный дизайн

### Описание основной задачи и требования

- [Подробное описание](christmas-shop.md)

В этой части задачи мы адаптируем ранее созданные страницы (**Главная** и **Подарки**) для планшетов и мобильных устройств.
Цель — соответствие макета соответствующему [Дизайну в Figma](https://www.figma.com/design/zTB01BwWZVoXYK5atH3eZT/Cristmas-Shop) при разрешениях экрана 1440px, 768px и 380px.
Выбор контрольных точек (breakpoints) и их количества остается на усмотрение студента.
Для всех остальных разрешений вплоть до 380px включительно обеспечьте:

- Отсутствие горизонтальной полосы прокрутки.
- Адаптивность и отзывчивость макета.
- Сохранение всего содержимого страницы.
- Сохранение пропорций изображений.
- Отсутствие пустых областей справа от секций.

Для создания адаптивного дизайна используйте относительные единицы измерения (%, rem, vh и т.д.).

## Порядок работы

1. Задача должна быть выполнена в приватном репозитории школы. [Как работать с приватным репозиторием школы](https://rs.school/docs/ru/private-repository).
2. Создайте новую ветку с именем `christmas-shop-part2` от ветки `christmas-shop`. В этой ветке вы найдете папку `christmas-shop`, содержащую файлы проекта, выполненные на предыдущем этапе.
3. Продолжайте работу над задачей в созданной вами ветке.
4. Убедитесь, что ваша работа соответствует [Критериям перекрестной проверки](#критерии-перекрестной-проверки).
5. Самостоятельно оцените свою работу в соответствии с предоставленными [Критериями оценки](#критерии-оценки), принимая во внимание [Детали проверки в инструментах разработчика](#детали-проверки-в-инструментах-разработчика) и [Особенности проверки верстки](#особенности-проверки-верстки).
6. Создайте Pull Request из ветки `christmas-shop-part2` в ветку `christmas-shop`, разрешите возникшие конфликты, а затем выполните слияние Pull Request.
   Название Pull Request может быть выбрано на ваше усмотрение. Описание для этого Pull Request не требуется.
7. После завершения перекрестной проверки предыдущей части задачи обновите деплой вашего проекта. Например, вы можете создать и объединить Pull Request из ветки `christmas-shop` в ветку `gh-pages`.
   Название Pull Request может быть выбрано на ваше усмотрение. Описание для этого Pull Request не требуется.
8. После завершения работы обновите Pull Request из ветки `christmas-shop` в ветку `main`, чтобы включить новую функциональность из второй части задачи.
   Объединять этот Pull Request не требуется.
9. Отправьте ссылку на деплой вашего проекта в разделе "Cross-Check Submit" в [RS APP](https://app.rs.school/).
10. После истечения крайнего срока сдачи задачи начнется этап перекрестной проверки, который продлится три дня. Чтобы получить баллы за задачу, вы должны проверить все назначенные вам проекты и отправить результаты проверки в разделе "Cross-Check Review" в [RS APP](https://app.rs.school/).

## Детали проверки в инструментах разработчика

1. Чтобы открыть инструменты разработчика:
   - Нажмите клавишу `F12` или щелкните правой кнопкой мыши и выберите опцию `Исследовать элемент` (Inspect) в контекстном меню;
   - Нажмите на значок **Toggle Device Toolbar** в правом верхнем углу панели инструментов разработчика;
   - Выберите **Responsive** на верхней панели.
2. Убедитесь, что в режиме **Responsive** отсутствует вертикальная полоса прокрутки. Если полоса прокрутки присутствует, устраните ее:
   - Переключите тип устройства с `Desktop` на `Mobile` на панели Device Toolbar;
   - Если тип устройства не отображается, нажмите на три точки справа на панели Device Toolbar и выберите `Add device type`.
3. Установите ширину экрана в значение, указанное в описании задачи, чтобы проверить соответствие макета дизайну Figma. Если страница веб-сайта не перестраивается корректно или справа появляется белая область, возможно, потребуется обновить страницу несколько раз.
4. Сравните макет с дизайном на предмет точности.
5. Чтобы проверить адаптивность при различной ширине экрана, плавно изменяйте ширину экрана в инструментах разработчика от максимальной (1440px) до минимальной (380px) и убедитесь, что горизонтальная полоса прокрутки отсутствует при любой ширине. Если появляется полоса прокрутки или белая область справа, обновите страницу, так как макет мог обновиться некорректно.
6. При масштабировании экрана (например, увеличение до 125%) фактическая ширина может отличаться на 1-2 пикселя. Например, фактическое значение может быть 767px или 769px, даже если инструменты разработчика показывают 768px. Несмотря на это расхождение, настройтесь на точку перехода.
7. Убедитесь, что эффекты наведения на интерактивных элементах деактивированы для типа устройства `Mobile`. Нажмите на интерактивный элемент, чтобы убедиться, что он не остается в состоянии наведения. Затем переключитесь на тип устройства `Desktop`, чтобы проверить, что эффекты наведения работают.

## Особенности проверки верстки

- Допускается отклонение до 10px по горизонтали и вертикали от макета при условии сохранения визуального соответствия между дизайном и разметкой.
- Используйте расширение [PerfectPixel](https://chrome.google.com/webstore/detail/perfectpixel-by-welldonec/dkaagdgjmgdmbnecmcefdhjekcoceebi?hl=ru) для Chrome, чтобы проверить соответствие дизайну.
- При использовании расширения PerfectPixel убедитесь, что оно настроено на масштаб 1, а ваш браузер и операционная система также имеют масштаб 100%.
- Каждый блок и секция должны оцениваться независимо; недостатки в одном блоке не должны влиять на оценку последующих блоков. Начинайте проверку каждого нового блока с его выравнивания по наложенному макету из PerfectPixel.
- Для текста проверяйте его выравнивание и интервалы в пределах блока. Размеры текста следует проверять только по высоте. Вариации в ширине слов и межбуквенном интервале не считаются ошибками при условии использования правильного шрифта с указанными свойствами.
