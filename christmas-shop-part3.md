## Christmas Shop. Часть 3: Добавление функциональности

### Описание основной задачи и требования

- [Подробное описание](christmas-shop.md)

В этой части задачи вам необходимо добавить следующую функциональность на веб-страницы, созданные ранее в рамках задачи:

- Открытие и закрытие **бургер-меню** на страницах при ширине экрана 768px и менее.
- **Слайдер** на главной странице (`home`).
- **Таймер** на главной странице (`home`).
- **Случайные подарки** на главной странице (`home`).
- **Переключение категорий** для подарков на странице `gifts`.
- Кнопка **ВВЕРХ** на странице `gifts`.
- **Модальное окно** для выбранного товара на обеих страницах.

❗ Каждый DOM-объект (блок) с описанием подарка, будь то подарок в категории или модальное окно, будет генерироваться из объектов данных с соответствующими полями. Собранный JSON с необходимыми для проекта объектами можно найти в разделе [Материалы](#материалы).

## Порядок работы

1. Задача должна быть выполнена в приватном репозитории школы. [Как работать с приватным репозиторием школы](https://rs.school/docs/ru/private-repository).
2. Создайте новую ветку с именем `christmas-shop-part3` от ветки `christmas-shop`. В этой ветке вы найдете папку `christmas-shop`, содержащую файлы проекта, выполненные на предыдущем этапе.
3. Продолжайте работу над задачей в созданной вами ветке.
4. Убедитесь, что ваша работа соответствует [Критериям перекрестной проверки](#критерии-перекрестной-проверки).
5. Самостоятельно оцените свою работу в соответствии с предоставленными [Критериями оценки](#критерии-оценки).
6. Создайте Pull Request из ветки `christmas-shop-part3` в ветку `christmas-shop`, разрешите возникшие конфликты, а затем выполните слияние Pull Request.
   Название Pull Request может быть выбрано на ваше усмотрение. Описание для этого Pull Request не требуется.
7. После завершения перекрестной проверки предыдущей части задачи обновите деплой вашего проекта. Например, вы можете создать и объединить Pull Request из ветки `christmas-shop` в ветку `gh-pages`.
   Название Pull Request может быть выбрано на ваше усмотрение. Описание для этого Pull Request не требуется.
8. После завершения работы обновите Pull Request из ветки `christmas-shop` в ветку `main`, чтобы включить новую функциональность из третьей части задачи.
   Объединять этот Pull Request не требуется.
9. Отправьте ссылку на деплой вашего проекта в разделе "Cross-Check Submit" в [RS APP](https://app.rs.school/).
10. После истечения крайнего срока сдачи задачи начнется этап перекрестной проверки, который продлится три дня. Чтобы получить баллы за задачу, вы должны проверить все назначенные вам проекты и отправить результаты проверки в разделе "Cross-Check Review" в [RS APP](https://app.rs.school/).

### Критерии перекрестной проверки (90 баллов)

1. Реализация **бургер-меню** на обеих страницах: **+22**
   - При ширине страницы 768px и менее панель навигации скрывается, и появляется иконка бургера: **+2**
   - При клике на иконку бургера бургер-меню выезжает справа, а иконка бургера плавно превращается в крестик: **+2**
   - Бургер-меню занимает всю доступную область экрана под блоком `<header>`: **+2**
   - При клике на крестик бургер-меню плавно скрывается, уходя вправо за пределы экрана, а крестик плавно превращается в иконку бургера: **+2**
   - Иконка бургера создана с помощью HTML и CSS без использования изображений/svg: **+2**
   - Ссылки в бургер-меню работают, обеспечивая плавную прокрутку к якорным точкам: **+2**
   - При клике на любую ссылку (интерактивную или неинтерактивную) в меню бургер-меню скрывается, а крестик превращается в иконку бургера: **+2**
   - При клике на любую ссылку (интерактивную или неинтерактивную) в меню бургер-меню плавно скрывается вправо, а крестик плавно превращается в иконку бургера, если пользователь остается на той же странице: **+2**
   - Расположение и размеры элементов в бургер-меню соответствуют макету (горизонтальное и вертикальное центрирование пунктов меню): **+2**
   - Страница за открытым меню не прокручивается: **+2**
   - При увеличении ширины страницы более 768px иконка бургера и открытое бургер-меню скрываются, и появляется панель навигации: **+2**
2. Реализация **Слайдера** на главной странице (`home`): **+18**
   - Прокрутка слайдера в соответствующем направлении реализована нажатием кнопок со стрелками влево и вправо: **+2**
   - Кнопка со стрелкой влево неактивна в крайнем левом положении слайдера: **+2**
   - Кнопка со стрелкой вправо неактивна в крайнем правом положении слайдера: **+2**
   - Прокрутка слайдера сопровождается анимацией, похожей на карусель (способ выполнения анимации не оценивается): **+4**
   - Слайдер полностью прокручивается за 3 нажатия кнопки со стрелкой в одном направлении для ширины экрана более 768px и за 6 нажатий кнопки со стрелкой в одном направлении для ширины экрана 768px и менее: **+4**
   - При изменении ширины экрана слайдер возвращается в исходное положение, и слайдер может быть полностью прокручен с правильным количеством кликов соответствующей кнопки со стрелкой (работает без перезагрузки страницы): **+4**
3. Реализация **Таймера** на главной странице (`home`): **+8**
   - Таймер показывает правильное значение оставшихся дней, часов, минут и секунд до Нового года в часовом поясе `UTC+0`: **+2**
   - Ведущие нули не отображаются для однозначных чисел: **+2**
   - Таймер обновляется каждую секунду, отображая текущее оставшееся время с метками `days`, `hours`, `minutes`, `seconds` (метки не должны меняться): **+4**
4. При открытии или обновлении страницы в блоке **Best Gifts** на главной странице (`home`) отображаются 4 случайные карточки: **+4**
5. Реализация **Переключения категорий** для товаров на странице `gifts`: **+8**
   - Категория **ALL** активна, и все 36 подарков отображаются как при открытии, так и при перезагрузке страницы `gifts`. Порядок сортировки подарков не имеет значения: **+2**
   - При переключении категорий отображаются подарки выбранной категории. Порядок сортировки подарков не имеет значения: **+4**
   - Активна только вкладка выбранной категории: **+2**
6. Реализация кнопки **Прокрутки вверх** на странице `gifts`: **+12**
   - Кнопка может появляться только при ширине экрана 768px и менее: **+2**
   - В верхней части страницы кнопка не отображается: **+4**
   - Кнопка появляется после прокрутки страницы вниз на 300px: **+4**
   - При нажатии на кнопку страница прокручивается вверх: **+2**
7. Реализация **Модального окна** для выбранного подарка на обеих страницах: **+18**
   - Модальное окно с описанием конкретного подарка открывается при клике на любую часть карточки подарка: **+4**
   - Описание и суперспособности в модальном окне соответствуют выбранному подарку: **+4**
   - Часть страницы за пределами модального окна затемнена: **+2**
   - Когда модальное окно открыто, вертикальная прокрутка страницы становится неактивной; при закрытии снова становится активной: **+4**
   - Клик по области вокруг модального окна и кнопка **Закрыть** закрывают его: **+2**
   - Модальное окно центрировано по обеим осям, размеры элементов модального окна и их расположение соответствуют дизайну: **+2**
